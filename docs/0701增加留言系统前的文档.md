# PbootCMS-Go 開發技術文檔（留言系統前基線版）

> 本文檔記錄留言系統開發前的項目技術全貌，供新人接手參考。
> 對應源碼版本：PbootCMS 3.2.12 PHP → Go 移植版
> 最後更新：2026-07-01

---

## 一、項目概述

PbootCMS-Go 是將 PHP 版 PbootCMS 3.2.12 忠實移植為 Go 語言的企業網站管理系統。項目保留了原版的數據庫表結構（`ay_` 前綴）、模板語法、URL 路由規則和後台 UI（Layui），用 Go 技術棧替換了 PHP 後端。

### 技術棧一覽

| 層次 | 技術 | 說明 |
|------|------|------|
| Web 框架 | Gin v1.12.0 | 路由分發、中間件、請求處理 |
| ORM | GORM v1.31.1 | 數據庫操作，AutoMigrate |
| 數據庫 | SQLite（glebarez/sqlite） | 純 Go 驅動，無需 CGO，文件 `data/pbootcms.db` |
| 後台模板 | Pongo2 v6.1.0 | Django 風格模板 + 自研 PbootCMS 語法轉換器 |
| 前台模板 | 自研 TagParser | `{gboot:xxx}` 標籤語法 + fsnotify 熱重載 |
| 前端 UI | Layui 2.5.4 + jQuery 1.12.4 | 後台界面，與 PbootCMS 原版一致 |
| 前台 UI | Bootstrap 4 + Swiper 4 | 前台模板自帶 |
| Go 版本 | 1.25.0 | |

### 核心設計原則

1. **數據庫零改動**：所有表結構、欄位名、表前綴 `ay_` 與 PbootCMS PHP 版完全一致，可直接遷移數據
2. **模板語法兼容**：後台保留 PbootCMS 原版 PHP 模板語法（`{foreach}`、`{if}`、`{$var->field}`），前台使用 `{gboot:xxx}` 標籤
3. **URL 100% 兼容**：通過路由重寫和 NoRoute 兜底，實現原版 URL 格式的完整支持

## 三、啟動流程

`main.go` 的 `main()` 函數按以下順序執行：

```
1. config.Load("config/config.json")     → 加載配置（sync.Once 單例）
2. model.InitDB(cfg)                     → 初始化數據庫（GORM + SQLite）
3. system.AutoMigrate()                  → 系統模型自動遷移（11個表）
4. content.AutoMigrate()                 → 內容模型自動遷移（14個表）
5. member.AutoMigrate()                  → 會員模型自動遷移（4個表）
6. seed.Init()                           → 種子數據初始化（冪等）
7. basic.InitViewEngine(...)             → 初始化後台模板引擎（pongo2）
8. parser.NewTemplateStore(...)          → 初始化前台模板引擎（加載+熱重載）
9. gin.Default() + 中間件 + 路由註冊     → 創建 Gin 引擎
10. r.Run(":8080")                       → 啟動 HTTP 服務
```

### 種子數據

`seed.Init()` 在首次運行時寫入：
- 管理員賬號：`admin` / `admin`（密碼雙重 MD5：`md5(md5("admin"))`）
- 站點信息、公司信息
- 36 條後台菜單（6個一級菜單 + 30個子菜單，與 PbootCMS 3.2.12 原版 1:1 對齊）
- 超級管理員角色 `R100`
- 2個會員組、2個內容模型
- 12項系統配置

## 四、數據庫層

### 連接配置

```go
// core/db/db.go
DB, err = gorm.Open(sqlite.Open(dsn), &gorm.Config{
    NamingStrategy: schema.NamingStrategy{
        TablePrefix:   "ay_",        // 保留 PbootCMS 原版表前綴
        SingularTable: true,          // 單數表名（user 而非 users）
    },
})
sqlDB.SetMaxOpenConns(1)              // SQLite 紅線：單連接防鎖死
sqlDB.SetMaxIdleConns(1)
sqlDB.SetConnMaxLifetime(5 * time.Minute)
```

### 模型組織

29個 GORM 模型按三個子包組織，通過 `apps/admin/model/db.go` 的 type alias 統一導出：

```go
// apps/admin/model/db.go
var DB = db.DB                        // 全局數據庫實例
type Content = content.Content        // 類型別名，控制器只需 import model
type Message = content.Message
type AdminUser = system.AdminUser
// ... 共 29 個別名
```

**系統模型（11個）**：AdminUser、Menu、MenuAction、Role、RoleLevel、Syslog、Area、Config、Type、DictType、DatabaseModel

**內容模型（14個）**：Company、Content、ContentSort、ExtField、Form、FormField、Label、Link、Message、Model、Single、Site、Slide、Tags、MediaMark

**會員模型（4個）**：Member、Comment、MemberGroup、MemberField

## 五、路由系統

### 前台路由

直接註冊在 Gin Engine 上，無中間件：

| 方法 | 路徑 | 控制器方法 | 說明 |
|------|------|-----------|------|
| GET | `/` | `fc.Index` | 首頁 |
| GET | `/search` | `fc.Search` | 搜索頁（支持分頁） |
| GET | `/tags` | `fc.Tags` | 標籗頁 |
| GET/POST | `/message` | `fc.Message` | 留言頁 |
| GET | `/api/visits` | `fc.Visits` | 訪問量統計 |
| GET | `/api/checkcode` | `fc.CheckCode` | 驗證碼圖片 |
| GET | `/sort/:scode` | `fc.SortByScode` | 欄目列表頁 |
| GET | `/content/:id` | `fc.ContentByID` | 內容詳情頁 |
| NoRoute | `*` | `fc.ContentPage` | 動態內容頁兜底 |

### 後台路由

統一前綴 `/admin`，掛載 `AdminAuth()` 中間件。在 `apps/route/route.go` 中集中註冊，按模塊分組：

- **IndexController** — 登錄、首頁、上傳、驗證碼
- **content 模塊** — 內容/欄目/單頁/幻燈片/友鏈/留言/標籤/表單/模型/擴展字段/媒體庫
- **system 模塊** — 菜單/用戶/角色/配置/數據庫/區域/日誌/類型
- **member 模塊** — 會員/會員組/會員字段/評論

### URL 重寫機制

由於 PbootCMS 原版 PHP 模板生成的 URL 與 Go 路由樹不完全匹配，`NoRoute` 實現三層兜底：

1. **大小寫歸一化**：`/admin/Content/index` → `/admin/content/index`
2. **PbootCMS 短路徑重寫**：`/admin/config` → `/admin/system/config`（30+ 條映射規則）
3. **前台內容頁兜底**：基於 `filename`/`urlname` 的偽靜態 URL 解析

### .html 後綴剝離

全局中間件：所有以 `.html` 結尾的請求剝離後綴後重新路由，使 `/content/7.html` 和 `/content/7` 訪問同一頁面。

## 六、後台管理系統

### 認證流程

**Session 機制**（`apps/common/session.go`）：
- 自實現內存 Session（`map[string]SessionData` + `sync.RWMutex`），非持久化，重啟丟失
- Cookie 名 `PbootGo`，有效期 24 小時

**登錄流程**（`IndexController.Login`）：
1. 驗證碼校驗（算術驗證碼，答案存內存 map）
2. 登錄鎖定檢查（同 IP 失敗 5 次鎖 900 秒）
3. 密碼雙重 MD5：`md5(md5(明文))`
4. 查詢 `ay_user` 表驗證
5. Session 寫入 uid/username/realname/ucode/rcodes/levels

**認證中間件**（`middleware/auth.go`）：
- 白名單路徑直接放行：`/admin/`、`/admin/index/login`、`/admin/index/checkCode`
- 未登錄：AJAX 返回 JSON，普通請求 302 重定向
- 已登錄：Session 信息注入 `gin.Context`

### BaseController 通用方法

所有後台控制器通過內嵌 `common.BaseController` 復用：

| 方法 | 功能 |
|------|------|
| `BatchSort(c, modelPtr, sortCol, defaultSort)` | 通用批量排序（髒檢查優化） |
| `JSONOK(c, data)` | 成功響應 `{"code":1,"data":...}` |
| `JSONOKMsg(c, msg)` | 成功響應帶消息 |
| `JSONFail(c, msg)` | 失敗響應 `{"code":0,"msg":...}` |
| `GetAdminUID(c)` / `GetAdminUsername(c)` | 從 Session 獲取用戶信息 |
| `IsLogin(c)` | 判斷登錄態 |

### 配置讀寫機制

- **讀**：`model.GetConfigValue(name, defaultVal)` — 查 `ay_config` 表
- **渲染注入**：`Render.go` 的 `loadConfigToData()` 將全部配置注入模板數據
- **寫**：`ConfigController.Index` POST 時遍歷預定義配置項數組逐個 upsert

### 通知消息常量

`apps/common/notice.go` 統一管理所有提示文案（繁體中文），項目硬約束：禁止硬編碼字符串，必須引用此處常量。

## 七、後台模板引擎

### PbootCMS 語法轉換器

後台使用 pongo2 模板引擎，但 PbootCMS 原版模板使用 PHP 語法。`core/basic/view.go` 的 `convertPbootToPongo2()` 函數通過 20+ 個正則表達式實現語法轉換：

| PbootCMS 語法 | 轉換後 pongo2 語法 |
|---------------|-------------------|
| `{include file='common/head.html'}` | 內聯展開（遞歸 5 層） |
| `{foreach $var(key,value)}` | `{% for valN in Var %}` |
| `{if($var->field==1)}` | `{% if Var.Field == 1 %}` |
| `{$var->field}` | `{{ Var.Field }}` |
| `{url./admin/path}` | 小寫化路徑 |
| `{$session.xxx}` | `{{ SessionXxx }}` |
| `{$get.xxx}` | `{{ GetXxx }}` |
| `{CMSNAME}` | `{{ CmsName }}` |
| `{php}...{/php}` | 清除 |

### 命名轉換

`flattenData()` 將所有數據 key 從 snake_case 轉為 PascalCase（含特殊縮寫 IP/URL/API/DB/CMS/HTML 及複合詞映射），使 PbootCMS 的 `{$config.page_size}` 能正確映射到 pongo2 的 `{{ Config.PageSize }}`。

### 自定義過濾器

`long2ip`、`safe`、`truncate`、`add`

### 模板渲染流程

`common.Render(c, tplPath, data)` 執行：
1. 注入 Session 信息（`session_xxx`）
2. 注入 CMS 常量（CmsName、CoreVersion、AppVersion、AppThemeDir 等）
3. 加載數據庫配置到 `Config` map
4. 注入 GET 參數（`get_xxx`）
5. 構建菜單樹（`MenuTree`、`MenuModels`）
6. snake_case → PascalCase 轉換
7. 獲取編譯緩存的 pongo2 模板並執行

### 後台模板組織

```
apps/admin/view/
├── common/          # 公共模板（head.html 含完整頭部+側邊欄，foot.html，ueditor.html）
├── content/         # 內容管理模板（17個）
├── system/          # 系統管理模板（13個）
├── member/          # 會員管理模板（4個）

```

## 八、前台模板引擎與標籤系統

### 雙引擎架構

項目使用雙模板引擎：後台用 pongo2 + 語法轉換器，前台用自研 TagParser + TemplateStore。

### TemplateStore（模板存儲與熱重載）

- 啟動時全量加載 `template/default/` 下所有 `.html`/`.htm` 文件到內存
- 使用 `fsnotify` 監聽文件變更，實現模板熱重載
- `Render(name)` 方法從內存取出模板 + 遞歸解析 `{include file=xxx}`
- 讀寫使用 `sync.RWMutex` 保證線程安全

### TagParser 解析流水線

每次請求新建 `TagParser` 實例，按以下固定順序執行：

1. 提取 `{gboot:pre}...{/gboot:pre}` 保護塊（暫存 JS/CSS 中的花括號）
2. 處理 include
3. 預解析配對標籤參數中的單標籤（如 `{gboot:list scode={sort:scode}}`）
4. 深度預解析嵌套 `{gboot:xxx}` 單標籤（最多 10 次迭代）
5. 處理配對標籤（list/nav/slide/link 等）
6. 處理單標籤（site/sort/content/page 等）
7. 處理 if 條件標籤
8. 恢復 pre 保護塊

### 標籤語法規範

**單標籤**：`{site:title}`、`{gboot:sitetitle}`、`{sort:name}`、`{content:title}`

**配對標籤**：
```
{gboot:list scode=5 num=10}
    [list:title]
    [list:link]
{/gboot:list}
```
內部標籤用方括號 `[prefix:field]`，每條記錄有 `[prefix:i]`（1-based）和 `[prefix:n]`（0-based）序號。

**條件標籤**：
```
{gboot:if(condition)}
    true 分支
{else}
    false 分支
{/gboot:if}
```
支持 4 層嵌套（`{gboot:if}`、`{gboot:1if}`、`{gboot:2if}`、`{gboot:3if}`），運算符：`==` `!=` `>` `<` `>=` `<=` `%`（非空判斷）和 `&&` `||`。

### 已註冊標籤清單

**單標籤**：site、gboot（兼容前綴）、company、sort、content、page、user、label、qrcode、position、pagetitle、pagekeywords、pagedescription、httpurl、pageurl、islogin、checkcode、msgaction、scaction、form、keyword

**配對標籤**：list、nav、sort_loop、content_loop、slide、link、loop、tags、pics、checkbox、message、formlist、search、selectall、select

### 內部標籤參數修飾

| 參數 | 示例 | 說明 |
|------|------|------|
| `len=N` | `[list:title len=12]` | 按字符截斷 |
| `lencn=N` | `[list:title lencn=30]` | 按中文寬度截斷 |
| `drophtml=1` | `[list:content drophtml=1]` | 去除 HTML 標籤 |
| `style=Y-m-d` | `[list:date style=Y-m-d]` | 日期格式化 |
| `decode=1` | `[content:content decode=1]` | HTML 實體解碼 |
| `substr=0,10` | `[content:title substr=0,10]` | 子字符串 |

### Context 結構體

```go
type Context struct {
    Sort        *model.ContentSort      // 當前欄目
    Content     *model.Content          // 當前內容
    Site        *model.Site             // 站點配置
    Company     *model.Company          // 公司信息
    Page        map[string]interface{}  // 分頁信息
    Member      *model.Member           // 會員（暫未實現）
    Keyword     string                  // 搜索關鍵詞
    CurrentPage int                     // 當前頁碼
    Filters     map[string]string       // ext_ 篩選參數
}
```

`buildContext()` 每次請求執行：初始化 Page/Filters → 收集 `ext_` URL 參數 → 從數據庫加載 Site 和 Company。

### 分頁機制

- 模板中 `page=1` 參數啟用分頁：`{gboot:list scode=5 num=15 page=1}`
- 頁碼通過 `?page=N` 傳遞
- 分頁標籤：`{page:numbar}`（數字分頁條）、`{page:index}`、`{page:pre}`、`{page:next}`、`{page:last}`
- 分頁 basePath 保留所有 `ext_` 篩選參數
- 分頁模板：`{include file=comm/page.html}`

## 九、前台路由與模板映射

### URL 生成規則

**欄目連結**（sortToMap）：
1. `outlink`（外部連結）優先
2. `filename` → `/filename/`
3. `urlname` → `/urlname/`
4. fallback → `/sort/scode`

**內容連結**（contentToMap）：
1. `outlink` 優先
2. `filename` → `/filename`
3. fallback → `/content/{id}`

### ContentPage 兜底邏輯

NoRoute 處理基於 filename/urlname 的偽靜態 URL：
1. 去除 `.html`/`.htm` 後綴
2. 優先查 `content.filename` → fallback `content.urlname`（status=1）→ 內容詳情頁
3. 否則查 `sort.filename` → fallback `sort.urlname` → 欄目列表頁
4. 都不命中 → 404

### renderSortPage 模板選擇

- 查 `ay_model` 表的 `type` 字段
- `type == 1`（單頁模型）→ 使用 `sort.ContentTpl`（如 about.html）
- 其他（列表模型）→ 使用 `sort.ListTpl`（如 list.html）
- 模板為空 → 默認 `list.html`

## 十、開發約束與規範

### 硬約束（必須遵守）

1. **數據庫欄位名**：使用 `mcode` 而非 `modelcode`；查詢自定義字段時 SQL 必須包含 `COALESCE(status,1) = 1` 處理 NULL 值
2. **Form action URL**：必須使用小寫 `/admin/content/mod`，大寫會導致 POST 數據丟失
3. **模板條件判斷**：字段類型必須用整數比較（`val1.Type==1`）而非字符串比較
4. **日期處理**：日期字符串必須用 `time.Parse("2006-01-02 15:04:05", v)` 解析後存入數據庫
5. **通知消息**：必須使用 `apps/common/notice.go` 中的常量，禁止硬編碼字符串
6. **編輯表單髒檢查**：無變更時不寫庫、不發成功通知
7. **構建產物**：必須輸出到 `bin/` 目錄
8. **文檔**：所有 `.md` 文檔放在 `docs/` 文件夾
9. **根目錄**：僅保留必要文件，臨時文件不得滯留
10. **首頁 Banner**：`.swiper-container`/`.swiper-wrapper`/`.swiper-slide` 最大高度 400px + `overflow: hidden`
11. **模板文件**：開頭的 BOM 字元（`U+FEFF`）必須清除
12. **導航欄**：使用 `sticky-top` 實現粘性置頂
13. **Layui checkbox**：多選字段不要使用 `lay-skin="primary"`

### 工程約定

- URL 參數中的 `&mcode=3` 需截斷 `&` 或 `?` 後綴提取有效 ID
- 多選框數據收集需同時檢查 `fieldName[]` 和 `fieldName[0]`、`fieldName[1]` 格式
- 零值 `time.Time` 顯示為空字符串
- Pongo2 模板使用控制器預格式化的 `DateStr`，不用不存在的 `date` 過濾器

### 經驗教訓

- Layui form 模塊會自動將 `[]` 名稱的 checkbox 重新索引為 `fieldName[0]`、`fieldName[1]` 格式
- 存儲未解析的日期字符串到 `time.Time` 欄位會顯示零值日期（0001-01-01）
- Layui 表單中帶 `lay-submit` 的排序按鈕需設置 `lay-submit` 屬性防止表單攔截
