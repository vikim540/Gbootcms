#### ✅ Task 1:Render.go - 注入 GET 參數 + backurl/pathinfo/btnqs

**實證文件**:`apps/common/Render.go:50-72`

**代碼證據**:
```go
// Inject GET query parameters for template access (e.g. {$get.mcode} → Get_mcode after flatten)
for key, values := range c.Request.URL.Query() {
    data["get_"+key] = values[0]
}

// Inject backurl / pathinfo / btnqs (used by PbootCMS mod-form action URLs)
qParams := c.Request.URL.Query()
var backParts, btnParts []string
var pathinfoBuilder strings.Builder
for key, values := range qParams {
    v := values[0]
    backParts = append(backParts, key+"="+v)
    btnParts = append(btnParts, key+"="+v)
    pathinfoBuilder.WriteString(fmt.Sprintf(`<input type="hidden" name="%s" value="%s">`, key, html.EscapeString(v)))
}
if len(backParts) > 0 {
    data["backurl"] = "&" + strings.Join(backParts, "&")
    data["btnqs"] = "?" + strings.Join(btnParts, "&")
} else {
    data["backurl"] = ""
    data["btnqs"] = ""
}
data["pathinfo"] = pongo2.AsSafeValue(pathinfoBuilder.String())
```

**結論**:✅ **完全屬實**。4 個數據注入(`get_xxx` 系列、`backurl`、`pathinfo`、`btnqs`)全部就位,且做了 `html.EscapeString` XSS 防護。代碼質量高。

## 三、業務代碼行業標準評估

### 3.1 評估方法

基於已讀源碼,從**專業架構師**視角評估,**不誇張、不挑刺**。
對標框架:Spring Boot(Java)、Laravel(PHP)、Django(Python)。

### 3.2 ✅ 優秀的設計(業界標準)

#### 3.2.1 Render 中 GET 參數注入(最佳實踐)

```go
for key, values := range c.Request.URL.Query() {
    data["get_"+key] = values[0]  // ← 前綴化變量,避免與業務變量衝突
}
```

**優點**:
- ✅ 在單一位置(`Render()`)集中處理,所有 Controller 自動受益
- ✅ 使用 `get_xxx` 前綴,避免與 `gin.H` 中其它業務 key 衝突
- ✅ pongo2 模板可直接訪問 `{{ get_mcode }}`,無需 transpiler

**業界對標**:與 Laravel `$request->all()`、Spring `@RequestParam` 同等。

---

#### 3.2.2 pathinfo 構造 + XSS 防護(安全意識)

```go
pathinfoBuilder.WriteString(fmt.Sprintf(`<input type="hidden" name="%s" value="%s">`,
    key, html.EscapeString(v)))
data["pathinfo"] = pongo2.AsSafeValue(pathinfoBuilder.String())
```

**優點**:
- ✅ `html.EscapeString` 防 XSS
- ✅ `pongo2.AsSafeValue()` 告訴 pongo2「這是安全 HTML,不要再次 escape」

**業界對標**:與 Spring `Thymeleaf` `th:utext` 等價行為。

---

#### 3.2.3 Controller-Service-Helper 三層分離(架構規範)

```
Controller (HTTP 入口) → Service (業務邏輯) → Model (數據訪問)
                                ↓
                          Helper (通用工具)
```

**優點**:
- ✅ Controller 變薄(只做「收參數 → 調 Service → 渲染」)
- ✅ Helper 抽離通用邏輯(`AddSortName`, `GetAllModelsData` 等)
- ✅ Service 層職責清晰

**業界對標**:與 Spring Boot `@Controller` + `@Service` + `@Component` 等價。
