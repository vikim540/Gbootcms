# PbootCMS Go 重构完善计划

## Context

当前项目已完成 PHP→Go 的大部分迁移，目录结构与 PbootCMS 原版对齐，控制器/模型/路由均已搭建。但存在一个**致命问题**：项目**无法编译**，因为存在循环导入（`model/seed.go` → `model/content` → `model`）。此外还有缺失的方法、类型别名、初始化调用等需要修复。

## 已发现的问题清单

1. **循环导入**：`model/seed.go` 导入 `model/content`、`model/system`、`model/member`，而这些子包又导入 `model` 获取 DB
2. **类型引用断裂**：控制器用 `model.Config` 等引用类型，但类型定义在子包中，`model` 包无别名
3. **缺失方法**：`BaseController.JSONFailMsg` 被多个控制器调用但未定义
4. **缺失函数**：`model.GetConfigValue`、`model.GetDBName`、`model.TableName` 未定义
5. **content/db.go 空指针**：`var DB = model.DB` 在 init 时是 nil
6. **AutoMigrate 不完整**：仅调用了 system，缺少 content 和 member
7. **SeedData 未调用**：种子函数从未在 main.go 中调用
8. **表名不对齐**：system 子包的 TableName() 缺少 `ay_` 前缀

## Task 1: 修复循环导入 — 将 seed 逻辑移到独立包

## Task 6: 统一 pongo2 渲染引擎

**当前问题**: 项目有两套渲染引擎并存：
- `core/basic/view.go`（pongo2）— 仅 Home 页面使用
- `apps/common/Render.go`（811行正则）— 其他所有后台页面使用

**目标**: 统一使用 pongo2 渲染所有后台页面。

**方案**:
- 重构 `common.Render()` 使其调用 `basic.GetAdminView()` + `basic.RenderAdminView()`
- 保留 `Render.go` 中的数据准备逻辑（菜单树、配置加载、session注入），删除正则模板处理部分
- 确保 `convertPbootToPongo2()` 能正确处理所有后台模板语法
- 逐个验证后台模板渲染正确性

**关键文件**:
- `apps/common/Render.go` — 重构为 pongo2 调用
- `core/basic/view.go` — 可能需要扩展模板转换规则
- 各控制器中的 Render 调用无需改变（接口不变）

### Task 9：前端 UX 修復 — 菜單高亮 + 表單反饋

**菜單高亮修復（3 個文件）：**
- `main.go`：NoRoute 中 `c.Request.Header.Set("X-Original-Path", original)` 保存原始 URL
  - **關鍵發現**：Gin v1.12 的 `HandleContext()` 調用 `c.reset()` 會清空 `c.Keys`，因此不能用 `c.Set()`，改用 HTTP 請求頭
- `apps/common/Render.go`：
  - `data["URL"]` 改為從 `X-Original-Path` 請求頭讀取（PbootCMS 原始 URL）
  - `data["C"]` 改為 `extractController(pageURL)` 動態提取（不再硬編碼 `"index"`）
  - 新增 `extractController()` 函數：`/admin/company/index` → `"company"`
- `static/admin/js/comm.js`：`light_nav()` 第四策略恢復工作（C 不再硬編碼）

**表單 AJAX 反饋（3 個文件）：**
- `static/admin/js/mylayui.js`：添加 `$(document).on('submit', 'form.layui-form:not(#dologin)', ...)` 通用 AJAX 處理器
  - **關鍵發現**：layui v2.5.4 的 `form.on('submit(*)', ...)` 通配符不生效，改用 jQuery 事件委託
- `templates/admin/content/company.html`：`<form>` 添加 `class="layui-form"`
- `templates/admin/system/database.html`：`<form>` 添加 `class="layui-form"`
- 成功時 `layer.msg()` 彈出 toast，不再顯示原始 JSON
